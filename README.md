## 🚀 Установка и запуск

### Требования

- Node.js (версия 18 или выше)
- PostgreSQL (локально или облачная БД)

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/neon-jump.git
cd neon-jump
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Настройка базы данных

**Вариант А: Использование существующей облачной БД (Render.com)**

В файле `server.js` уже настроено подключение. Просто убедитесь, что база данных активна.

**Вариант Б: Локальная PostgreSQL**

1. Создайте базу данных:
```sql
CREATE DATABASE neon_jump;
```

2. Измените `connectionString` в `server.js`:
```javascript
const pool = new Pool({
  connectionString: 'postgresql://localhost:5432/neon_jump',
  user: 'your_username',
  password: 'your_password',
  ssl: false
});
```

### 4. Настройка JWT секрета

В `server.js` измените строку:
```javascript
const JWT_SECRET = 'your-secret-key-change-in-production';
```
На надёжный секрет:
```javascript
const JWT_SECRET = process.env.JWT_SECRET || 'super-secret-key-2025';
```

### 5. Запуск сервера

```bash
npm start
```

Сервер запустится на `http://localhost:3000`

### 6. Игра

Откройте браузер и перейдите по адресу `http://localhost:3000`. Зарегистрируйтесь или войдите в существующий аккаунт и начинайте играть!

## 🏗️ Архитектура и механики

### Серверная часть (`server.js`)

#### Основные компоненты:

1. **Express сервер** — обрабатывает HTTP запросы (регистрация, логин, лидерборд)
2. **Socket.IO** — управляет realtime соединениями (движение, прыжки, чат)
3. **PostgreSQL Pool** — пул соединений с базой данных
4. **ActiveUsers Map** — отслеживает активных пользователей (userId → socketId)
5. **Players Map** — хранит данные всех игроков в памяти

#### Основные механики:

- **Аутентификация через токены:** Каждый Socket.IO запрос требует JWT токен для идентификации
- **Активные сессии:** При повторном входе старый сокет автоматически отключается
- **Награда за прыжки:** `coinsReward = jumpHeight × 5`
- **Улучшение прыжка:** `jump_power += 2`, стоимость = `(старая сила) × 10`
- **Корона лидера:** Проверяется после каждого изменения монет

### Клиентская часть (`game.js`)

#### Трёхмерный мир:

- **Сцена:** Neon-стиль, тёмный фон с неоновой подсветкой
- **Камера:** Следует за игроком сверху-сзади
- **Персонажи:** Кубы 2×2×2 с неоновыми материалами
- **Освещение:** 3 точечных источника света (cyan, magenta, yellow)
- **Платформа:** 100×100 единиц с сеткой и свечением

#### Физика движения:

- **Скорость:** 0.5 единиц на кадр
- **Гравитация:** -0.3 на кадр
- **Сила прыжка:** `jump_power / 10` (начальная 30 → сила 3)
- **Высота прыжка:** Отслеживается максимальное значение Y во время прыжка

#### UI компоненты:

- **Форма авторизации:** Логин и регистрация
- **Игровой интерфейс:** Монеты, сила прыжка
- **Таблица лидеров:** Топ-10 игроков
- **Чат:** Общение с другими игроками
- **Всплывающие уведомления:** +монеты, ошибки

## 📡 Socket.IO события

### От клиента к серверу

| Событие | Данные | Описание |
|---------|--------|----------|
| `authenticate` | `token` | Аутентификация пользователя |
| `move` | `{x, y, z}` | Обновление позиции игрока |
| `jumpLand` | `jumpHeight` | Приземление после прыжка |
| `upgradeJump` | - | Улучшение силы прыжка |
| `chatMessage` | `message` | Отправка сообщения в чат |

### От сервера к клиенту

| Событие | Данные | Описание |
|---------|--------|----------|
| `init` | `{players, myId, topPlayerId}` | Инициализация игры |
| `playerJoined` | `playerData` | Новый игрок подключился |
| `playerMoved` | `playerData` | Игрок переместился |
| `playerLeft` | `playerId` | Игрок отключился |
| `coinsEarned` | `{coinsReward, user}` | Игрок получил монеты |
| `upgradeSuccess` | `user` | Улучшение успешно |
| `upgradeFail` | `message` | Ошибка улучшения |
| `crownUpdate` | `topUserId` | Обновление короны лидера |
| `chatMessage` | `{username, message}` | Новое сообщение в чате |
| `playerUpdated` | `playerData` | Данные игрока обновлены |

## 🗄️ База данных

### Таблица `users`

| Колонка | Тип | Описание |
|---------|-----|----------|
| `id` | SERIAL PRIMARY KEY | Уникальный идентификатор |
| `username` | VARCHAR(50) UNIQUE | Имя пользователя |
| `password` | VARCHAR(255) | Хэш пароля (bcrypt) |
| `coins` | INTEGER DEFAULT 50 | Количество монет |
| `jump_power` | INTEGER DEFAULT 30 | Сила прыжка |
| `created_at` | TIMESTAMP | Дата регистрации |

### Примеры запросов:

```sql
-- Получение лидера
SELECT id FROM users ORDER BY coins DESC LIMIT 1;

-- Обновление монет после прыжка
UPDATE users SET coins = coins + $1 WHERE id = $2;

-- Улучшение прыжка
UPDATE users SET coins = coins - $1, jump_power = jump_power + 2 WHERE id = $2;
```

   - Валидируйте все входные данные
   - Настройте CORS правильно

3. **Масштабирование:**
   - Используйте Redis для Socket.IO адаптера
   - Добавьте кластеризацию Node.js
   - Настройте балансировку нагрузки

4. **Мониторинг:**
   - Логирование ошибок (winston, pino)
   - Мониторинг производительности
   - Метрики активных пользователей
